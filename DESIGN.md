## Design Guidelines — AttendSure

### Principle
Simple, clean, minimal UI is a hard requirement. Fast to use for both HR and Employee roles. Convey trust and credibility.

### Color
- Primary: a calm, trustworthy blue (#2563EB) for actions and key accents.
- Validity signals: green (#16A34A) for valid check-ins, red (#DC2626) for invalid — binary and unmistakable.
- Neutrals: white/near-white backgrounds (#FFFFFF, #F8FAFC), slate grays (#0F172A text, #64748B secondary).
- Generous whitespace; restrained use of color so validity status pops.

### Typography
- Clean sans-serif (system / Inter style). Clear hierarchy: large readable headings, comfortable body, legible labels.
- Numbers and status badges emphasized in the dashboard.

### Components
- Cards with subtle borders/shadows for grouping (dashboard rows, SOP entries).
- Prominent status badges (Valid / Invalid) with color + label, never color alone.
- Face-capture: a centered camera frame with a clear capture button and immediate, explicit result feedback.
- Dashboard: scannable table/list of employees with date, validity, and SLA indicators.
- Buttons: solid primary for main actions, ghost/secondary for the rest.

### Layout
- Role-appropriate simple navigation. Minimal chrome, content-forward.
- Mobile-friendly for employee check-in; comfortable desktop layout for HR dashboard.
